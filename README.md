# مدونة ماجد

موقع ثابت مستقل بالكامل — بدون قواعد بيانات، بدون اشتراكات، بدون أي خدمات خارجية.
الخطوط والصور مستضافة محلياً، والمحتوى ملفات نصية تملكها.

## البنية

```
المدونة/
├── build.py            ← المولّد (بايثون فقط، بدون مكتبات)
├── content/
│   ├── site.json       ← اسم الموقع، الروابط، صفحة «عني»
│   └── posts/          ← كل مقالة ملف .md مستقل
├── assets/             ← الخطوط والصور والتنسيق
└── docs/               ← الموقع الجاهز (يتولّد تلقائياً — لا تعدّله يدوياً)
```

## إدارة المحتوى عبر سانيتي (الطريقة الأساسية)

المحتوى يُدار من الاستوديو: **https://majed-blog.sanity.studio**

1. افتح الاستوديو وسجّل دخولك بحساب Google (majed.alahdal@gmail.com).
2. أضف «مقالة» جديدة واملأ الحقول (العنوان، الرابط اللاتيني، التصنيف، الحاسة، الصورة، النص…).
3. اضغط **Publish**.
4. الموقع يُعاد بناؤه تلقائياً كل ٦ ساعات، أو فوراً من
   GitHub ← Actions ← «إعادة بناء الموقع من سانيتي» ← Run workflow.
   ومحلياً: `python3 build.py` ثم commit و push.

المولّد يجلب المقالات من سانيتي تلقائياً (الإعداد في `content/sanity.json`)
وينزّل الصور محلياً فيبقى الموقع مستقلاً. لو حذفت `content/sanity.json`
يعود المولّد لملفات `content/posts/` المحلية.

## إضافة مقالة يدوياً (بديل بدون سانيتي)

1. أنشئ ملفاً في `content/posts/` باسم لاتيني قصير، مثلاً `madina-wa-dhakira.md`:

```
---
title: عنوان المقالة
date: 2026-08-01
category: تأمّل
subcategory: مشهد
reading_time: 6
image: assets/images/madina-wa-dhakira.png
featured: no
excerpt: وصف قصير يظهر في البطاقة وتحت العنوان.
pull_quote: اقتباس بارز يظهر في صفحة المقالة (اختياري).
---
<p>نص المقالة هنا. كل فقرة داخل وسم p.</p>

<h2>عنوان فرعي</h2>

<p>فقرة أخرى…</p>

<blockquote>اقتباس داخل النص.</blockquote>
```

2. ضع صورة المقالة في `assets/images/` بنفس الاسم.
3. شغّل: `python3 build.py`

- `featured: yes` تجعل المقالة تتصدّر الواجهة الرئيسية (اجعلها `yes` لمقالة واحدة فقط).
- التصنيفات: رأي / تأمّل / تجربة، والحواس: مشهد / طعم / صوت / رائحة / ملمس (تُعدَّل من `site.json`).

## المعاينة محلياً

```
python3 -m http.server 8788 --directory docs
```

ثم افتح http://localhost:8788

## النشر

مجلد `docs/` هو الموقع كاملاً — يصلح لأي استضافة ثابتة
(GitHub Pages، Cloudflare Pages، Netlify، أو أي خادم عادي).
إن أضفت رابط الموقع في `site.json` تحت `site_url` فسيتولّد ملف `feed.xml` (خلاصة RSS) تلقائياً.
